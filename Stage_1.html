Skills for Programmers

Meta Skills

Programming involves 5 meta skills. They are procedural thinking, abstract thinking, systems thinking, technological empathy, and debugging.

Procedural thinking is used to create unambiguous instructions for a computer to follow. It works in conjunction with systems thinking to break down a series of actions into smaller pieces that are easier to explain and code. These smaller pieces are then put together to create the complete solution for a problem.

Abstract thinking involves the generalization between things that appear to be different. It is the recognition of patterns that exist across domains. Mathematics is one form of abstract thinking. For example the derivative has applications in both physics and economics.

Systems thinking is used to break a problem down into it's component parts. It is used to slice and dice a problem down into smaller, more manageable pieces.

Technological Empathy is the ability to understand technology, both it's strengths and weaknesses. For programmers this means understanding computers, how they work, what they are good at, and what they aren't.

Debugging is the process of finding errors in a program. Both syntactic and semantic. It is the process a programmer uses to eliminate grammatical and logical errors from a program (failures to correctly communicate what a programmer wants to a computer). Computers are stupid, they cannot make guesses about a programmer's intended meaning. They can only execute the literal instructions that they were given.

Web Basics

Basics of the Web and HTML

The web consists of several major components. They are the end user, their computer and it's browser, the internet, and the servers that host content and direct traffic.

HTML (Hyper Text Markup Language) is the glue that holds a web page together. It describes and creates the structure of a web page. It can contain text, markup, references to other documents, and links to other pages.

Hyperlinks link one page to another.

HTML Markup

HTML documents are made up of elements. They are the building blocks that, when combined, create the structure of a web page.

An element usually consists of an opening tag and closing tags. The opening tags often contain attributes that define some property of the element. Void tags, such as the line break tag, do not require a closing tag.

Elements can be inline or block. Inline elements exist on a line while block elements create a box that will hold the content.

Span is an example of an inline element and div is an example of a block element. Each can take CSS class attributes that will define how they appear.

Two very important types of attributes are the class and ID attributes. Both act as hooks that CSS and JavaScript can use to identify specific elements or groups of elements so that they can act on a web page.

Class attributes identify elements as an instance of a given class. They group elements together by style. Using them effectively can both drastically reduce the amount of code that needs to be written and the amount of bugs on a page. It also makes it much easier to modify and debug a page. It enables developers to create a web page without repeating themselves.

It does this by allowing a developer to write one style definition that can then be applied to all elements that have that style attribute. All elements affected by that style will be affected by any changes made to it. Less code means less typos and other bugs when it is initially coded. It means less code to read and correct when debugging, and it reduces the opportunities for bugs to sneak in while it is being modified. Fewer bugs is a natural consequence of there being only one section of code that needs to be modified, instead of two, three, ten, or more to affect a change.

ID attributes act as unique identifiers. They allow a developer to target a single element for modification by CSS or Javascript, as well as enabling the creation of links within a page. They must be unique or the file will not validate. The ability to use them as targets within a page is also dependent upon their uniqueness. If something needs modify more than a single element, use a class.

A single element can have multiple classes. They are space delimited.

Structure of an HTML Document - Part 1

Every HTML document has several structural elements. The first is the Doctype, it is followed by the Root of the document, defined by the HTML tag. The HTML tag contains head and body elements.

The doctype element tells the browser what version of HTML the page uses.

The root (HTML) element contains all the content of the page. All other elements grow out of this. It is the base of the tree from which all other elements grow. The opening tag comes just after the doctype declaration and it's closing tag comes at the end of the document.

The head contains metadata, JavaScript, and references to CSS style sheets. It's contents are not displayed.

The body contains all of the content and can be further divided into smaller sections using the span, div, section, article, and aside tags.

Interlude - Hints for Learning

From Novice to Expert

Students begin as Novices and pass through four stages on their way to becoming experts. They are Ignorance, Awareness, Ability, and Fluency.

Students can progress more quickly and easily through the process by taking the time to reflect on what they have learned. This helps them to take scattered facts and organize them into a structured understanding.

Practice is also extremely important if students are to gain the ability to execute on their good ideas.

There are some things to keep in mind when listening to experts. Sometimes they will assume that you have knowledge you don't, use jargon you don't understand, or speak too quickly for you to understand.

They are not trying to confuse you, they have just forgotten how hard it was for them when they were getting started. Ask questions when this happens.

Document Creation

Creating a Structured Document

The structure of a page is controlled by the DOM (Document Object Model). HTML defines the structure. While CSS defines the appearance of a page.

DOM is a cross platform and language independent convention for representing and interaction with objects in mark up languages such as HTML. It generally has a tree like structure with many branching elements emerging from the root.

HTML uses abox model. Each box has several layers. The central layer is the content of the box. It is enclosed by padding, a border, and a margin.

The content is the nucleus of the element and contains media of some kind, such as text or an image.

The padding clears an area around the content and is affected by the background color of the box.

The border goes around the padding and content and inherits the color property of the box.

The margin clears an area around the border and is completely transparent. It is the space between the boxes.

The element width is equal to the total width of the content, padding, and border. By setting the box sizing attribute to border-box the calculation on element size will include both the border and padding (not just the content). However, the margin is not included.

The border-box attribute is a relatively recent addition to CSS and so will not be supported by many older browsers. This can be fixed by adding browser specific prefixes.

Element size can be given in pixels or percentage of screen. If the size is set using pixels the element will not change size to accommodate itself to different screen sizes. If the element size is specified with percentages it will adapt itself automatically. Minimum and maximum screen sizes can also be specified.

The flex-box attribute will allow divs to appear next to each other. They are used so that a page can dynamically resize and rearrange itself in response to changes in size and orientation of the viewing area.

When choosing a tag to use to create an element, the function of the element should be considered. While many block elements are syntactically similar, they carry different semantic meanings. Heading, section, article, aside, and nav tags are all examples of block tags with special semantic meanings.

When creating a web page the Code, Test, Refine cycle is very effective. First the developer should consider the design of the page, looking for natural boxes and repeated styles and semantic elements. Next they should write their HTML code. Then apply styles working from the biggest structural elements of the page to the smallest. Finally they should go back and compare their results to the initial design, fixing any remaining minor details that may need adjustment.

HTML and CSS should be validated before being published to the web. Code that fails to validate fails to meet the standard specifications and may display differently than intended, depending on a given browser's response to the code the faulty code. Cascading Style Sheets

Cascading Style Sheets (CSS) are used to style a web page. The term cascading refers to the way a style applied to an element cascades down to all of it's children. In other words each child element inherits the CSS property values from their ancestor elements.

A style applied to a large branch will affect all of it's child branches. But that styling can be refined or overridden on any child branch by simply assigning another style to just that branch.

In general decedents inherit text-related properties, but not box ones.

A single HTML document can reference multiple CSS style sheets.
